Methodology

Here I describe some of the high level choices made and general processes taken to gather, clean, and visualize the data used in this project.

The Discogs database was used as the primary data source for the first two visualizations, which explore the geographic distribution of DnB releases over the past 30 years. The data was gathered using the Discogs API and a python client.

For the first visualization, the API was queried for releases with the style "Drum and Bass". These release objects had a country property which was geocoded to latitude longitude using the OpenCage Geocoding API. The resulting data was visualized using the Leaflet JavaScript library. To determine the top 5 releases to display, I utilized the "have" property of the release objects, which indicates how many users in the Discogs community have that release in their collection. I find this a reasonable metric as the top 5 releases is not meant to be an authoritative list but an approximate representation of some of the songs from that country.

For the second visualization, the API was queried for releases with the style "Drum and Bass" and the country "UK". The resulting release objects had their artist arrays parsed, and the profile of the first artist was checked for a matching city in the UK. These cities were again geocoded to latitude longitude using the OpenCage Geocoding API and visualized using the Leaflet JavaScript library. The reason this multi-step process was necessary was because no other resource provides a more specific geographic location for the releases, which unfortunately does lead to some artifacts in the data with Bristol not having very many releases.

The third visualization draws from the RapidAPI "Spotify Extended Audio Features API", which was queried for some hand-picked songs and their associated audio_features values. These values are ML-computed by Spotify and are not easily interpretable, but it sufficed to compare the differences between London and Bristol DnB songs. The reason I chose 5 songs from each city was to have a small enough sample size to be able to meaningfully cross-reference the data with the song feeling and feasibly make API requests. Given more time on the project, I would have liked to expand the sample size. The data was processed and visualized using the Chart.js JavaScript library.
